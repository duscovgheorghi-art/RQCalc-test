# Contributing to RQCalc-test

Welcome to the RQCalc-test! We appreciate your contributions to make this project better.

## Development Guidelines
- Ensure that you have a clear understanding of the project requirements before starting.
- Test your changes thoroughly before submitting a pull request.

## Code Standards
- Follow the [XYZ coding standard](link-to-coding-standard) for consistent code.
- Maintain proper indentation and spacing.
- Write clear and concise comments for easier code understanding.

## Commit Conventions
- Use present tense for commit messages. For example: "Fix bug in calculation."
- Include the issue number in the commit message if applicable. For example: "Fix #42: Correct calculation bug."
- Ensure that the commit message is no longer than 72 characters.

## Pull Request Process
1. Fork the repository and create your branch from `main`.
2. Make sure your code adheres to the above guidelines.
3. Open a pull request against the `main` branch of the original repository.
4. Describe your changes and provide any relevant context or links.
5. Wait for feedback and make adjustments as necessary.

Thank you for contributing!
